# Spec: Micro-Model First Inference

## Goal

Add cheap Level-1 intent classifier in front of existing multi-stage LLM pipeline. Early-exit on gibberish/out-of-scope phrases without calling gpt-4o-mini. Route only likely-valid phrases to Level 2.

## Motivation

From Day 9 routing benchmark: 37% of corpus escalates to gpt-4o, only 1/11 escalations actually change the outcome. Noisy group (`kakashki`, `xyzzy foo bar`, `Show ads`) consumes full LLM budget to produce `FAIL`. Gate these before the expensive call.

## Architecture

```
phrase
  ▼
[Level 1: MicroClassifier]
  - OpenAI text-embedding-3-small (1 API call, ~50ms)
  - cosine vs 3 prototypes (valid / gibberish / ambiguous)
  - returns {Intent, Confidence (top1 cos), Margin (top1-top2)}
  ▼
routing:
  - intent=gibberish  && margin≥τ_margin → early FAIL (skip LLM)
  - intent=ambiguous  || margin<τ_margin → Level 2 + self_check
  - intent=valid      && margin≥τ_margin → Level 2 direct
  ▼
[Level 2: MultiStageParser] (existing, unchanged)
  ▼
TwoLevelResult
```

## Intent Classes (3)

| Intent | Meaning | Corpus match |
|--------|---------|--------------|
| `valid` | Parseable into PublisherRule | correct group |
| `ambiguous` | Partially expressible / unclear scope | edge group |
| `gibberish` | Nonsense / out-of-scope / contradictory | noisy group |

## Prototypes

- Offline build test `TestBuildPrototypes` (skipped unless `OPENAI_API_KEY` set + explicit flag)
- Input: labeled corpus (reuse existing `corpus.jsonl`, derive label from `group`: `correct→valid`, `edge→ambiguous`, `noisy→gibberish`)
- Output: `finetune/confidence-check/data/prototypes.json` — map `intent → []float32` (mean embedding per class)
- Runtime load: embed JSON into Go via `//go:embed` or read-once on init

## API Surface

```go
// embed.go
type EmbedResponse struct {
    Vector       []float32
    InputTokens  int
}
type EmbedClient interface {
    Embed(ctx context.Context, text string) (EmbedResponse, error)
}
func NewOpenAIEmbedClient(apiKey, model string) EmbedClient  // default "text-embedding-3-small"

// micro.go
type Intent string
const (
    IntentValid     Intent = "valid"
    IntentAmbiguous Intent = "ambiguous"
    IntentGibberish Intent = "gibberish"
)

type MicroResult struct {
    Intent     Intent
    Top1       float64      // cosine to top prototype
    Margin     float64      // top1 - top2
    Scores     map[Intent]float64
    LatencyMS  float64
    InputTokens int
}

type MicroClassifier interface {
    Classify(ctx context.Context, phrase string) (MicroResult, error)
}

type EmbedMicro struct {
    client     EmbedClient
    prototypes map[Intent][]float32
}
func NewEmbedMicro(client EmbedClient, prototypes map[Intent][]float32) *EmbedMicro

// Helper exported for prototype builder.
func MeanVector(vecs [][]float32) []float32
func Cosine(a, b []float32) float64

// two_level.go
type Route string
const (
    RouteMicroEarlyFail Route = "micro_early_fail"
    RouteLLMDirect      Route = "llm_direct"
    RouteLLMWithCheck   Route = "llm_with_check"
)

type TwoLevelResult struct {
    RulesJSON    string
    Confidence   ConfidenceStatus
    Route        Route
    Micro        MicroResult
    LLM          *MultiStageResult  // nil when early-exit
    TotalLatency float64
    UsedLLM      bool
}

type TwoLevelParser struct {
    micro       MicroClassifier
    llm         *MultiStageParser
    marginFloor float64   // below this → LLMWithCheck
}
func NewTwoLevelParser(micro MicroClassifier, llm *MultiStageParser, marginFloor float64) *TwoLevelParser
func (p *TwoLevelParser) Parse(ctx context.Context, phrase string) TwoLevelResult
```

## Thresholds (initial)

- `marginFloor = 0.05` — below → treat as UNSURE, escalate with self-check
- Hard FAIL route: `intent=gibberish && margin≥0.05`

Tuning left to benchmark. First pass may adjust once distribution is observed.

## Benchmark

`microfirst_benchmark_test.go`:
1. Requires `OPENAI_API_KEY`. Skip otherwise.
2. Loads corpus via `loadCorpus(t)` (existing helper).
3. Loads prototypes from `finetune/confidence-check/data/prototypes.json`.
4. For each phrase:
   - Baseline: `MultiStageParser.Parse` (always Level 2).
   - Two-level: `TwoLevelParser.Parse`.
5. Counts per-route distribution, confidence distribution, latency, LLM calls saved.
6. Writes `report_microfirst.md`:
   - Per-phrase table (phrase / group / micro intent / margin / route / final conf / latency)
   - Summary: % handled by micro, LLM calls saved, accuracy vs expected_confidence
   - Delta vs baseline (gpt-4o-mini only, no routing)
