# Cloud vs Local LLM Comparison

## Task: Organizations CRUD (migration + sqlc + repository + service + handler)

| Критерий | Claude Code (Opus) | Local LLM (Qwen3 30B / Qwen2.5 Coder 14B) |
|----------|-------------------|-------------------------------------------|
| **Качество кода** | Высокое. Соблюдает все конвенции, правильные пути, правильная архитектура слоёв | Низкое. Неправильные пути, лишний код (триггеры), игнорирует системный промпт |
| **Скорость генерации** | ~30 сек (зависит от API) | ~2-5 мин (зависит от железа) |
| **Понимание контекста проекта** | Отличное. Читает CLAUDE.md, rules, существующий код, понимает структуру | Слабое. Видит только системный промпт, не понимает реальную структуру проекта |
| **Agent mode** | Полноценный. Создаёт файлы, запускает тесты, итерирует | Не работает. Tool calling сломан, JSON injection, chain-of-thought вместо action |
| **Следование правилам** | Строго следует CLAUDE.md и rules/ | Частично игнорирует системный промпт |
| **Работа без интернета** | Нет | Да |
| **Стоимость** | API tokens (платно) | Бесплатно (но требует мощное железо) |
| **Справилась с первого раза** | Да | Нет |
| **Multi-file generation** | Да, все слои за один запрос | Нет, генерирует по частям или ломает |
| **Автокомплит** | Нет (отдельный инструмент) | Да, встроен в IDE |

---

## Для каких задач локальная модель достаточна

### Подходит
- **Автокомплит** — дописывание строк, tab-completion (маленькие модели 1.5-3B)
- **Простые вопросы** — "что делает эта функция", "как работает этот паттерн"
- **Генерация бойлерплейта** — простые структуры, интерфейсы (в chat mode)
- **Работа без интернета** — самолёт, VPN-ограничения, приватный код
- **Приватность** — код не покидает машину

### Облако незаменимо
- **Agent mode** — создание файлов, запуск тестов, итерация по ошибкам
- **Сложные фичи** — multi-file CRUD, рефакторинг, архитектурные решения
- **Понимание контекста** — большие проекты, монорепо, кастомные конвенции
- **Следование правилам** — строгие code style, naming conventions, project rules
- **Code review** — анализ PR, поиск багов, security review

## Параметры генерации

| Параметр | Автокомплит | Chat/Agent |
|----------|-------------|------------|
| **temperature** | 0.0 (детерминированный) | 0.1 (минимальная вариативность) |
| **top_p** | 0.9 | 0.9 |
| **maxTokens** | 1024 | 4096 |
| **contextLength** | 8192 (по умолчанию) | 16384+ (для больших файлов) |

Для кода нужна **низкая temperature** (0.0–0.2) — высокая даёт галлюцинации и нестабильный вывод. top_p 0.9 — стандарт для всех задач.

## Лучшая связка для MacBook Pro 36GB RAM

### Финальная рекомендация

| Роль | Модель | Размер | Параметры |
|------|--------|--------|-----------|
| **Agent + Chat** | qwen3:30b-a3b (MoE) | 18 GB | temperature 0.1, maxTokens 4096 |
| **Автокомплит** | qwen2.5-coder:1.5b | 1.5 GB | temperature 0.0, maxTokens 1024 |
| **Embeddings** | nomic-embed-text | 274 MB | — |

Общий расход RAM: ~20 GB (остаётся запас для IDE и системы).

### Почему эта связка
- **qwen3:30b-a3b** — MoE архитектура, 3B активных параметров → быстрая как маленькая, качество как большая. Стабильный tool calling (по бенчмаркам)
- **qwen2.5-coder:1.5b** — специализирована на completion, быстрый отклик для tab-autocomplete
- **nomic-embed-text** — лёгкая, даёт семантический поиск по всему проекту

### Альтернативы для agent mode (если qwen3 не справится)
1. **devstral:24b** (~15 GB) — специально для SWE-агентов
2. **gpt-oss:20b** (~12 GB) — нативная поддержка Continue.dev agent mode

## Рекомендация

**Оптимальная связка:** облачный ассистент (Claude Code) для основной работы + локальная модель для автокомплита и работы офлайн.

Локальные модели в 2026 году **не заменяют** облачные для серьёзной разработки. Главные ограничения:
1. Agent mode не работает надёжно ни с одной локальной моделью < 30B
2. Контекст проекта теряется — модель не читает существующий код
3. Системный промпт игнорируется — правила и конвенции не соблюдаются

Для автокомплита и простых вопросов — локалка достаточна и экономит деньги.
